# LawLaw — 你的「案件工作台」说明书

这是一份给**第一次打开项目、不想先啃代码**的朋友看的说明。如果你要给 AI 或开发者讲技术细节，请看同目录下的 **[`AGENTS.md`](AGENTS.md)**。

---

## LawLaw 是干什么的？

可以把它想成：**一个只跑在你电脑上的小型律所工作台**。

- 左边管**项目**（每个项目就像「一个案子」的抽屉）。
- 中间是**法律意见书 / 文书**编辑器，还能按你上传的**参考模板**风格，让 AI **流式生成**一版草稿（证据会打包发给能看图的多模态模型）。
- 右边可以和 AI **聊天**：它不是瞎聊，会先在**当前项目**里上传的卷宗里**搜相关内容**，再回答（这叫 RAG，检索增强生成）。
- 底下的**证据池**用来上传 PDF、Word、压缩包、图片等；上传后后台会**切块、做向量**，方便以后检索。

**重要**：模型和 API 走的是**火山方舟**（豆包等），需要你自己的 **API Key** 和两个 **`ep-` 开头的接入点 ID**（一个给对话/视觉，一个给向量）。这些可以只在浏览器里填，也可以写在后端的 `.env` 里当默认值。

---

## 你需要准备什么？

1. 一台电脑，能装 **Node.js** 和 **Python 3**。
2. 火山引擎方舟账号：能创建 **API Key**、**Chat/Vision 推理接入点**、**Embedding 推理接入点**（都是 `ep-……` 这种格式）。
3. 十分钟左右耐心，按下面步骤点一点。

---

## 怎么跑起来？（四步走）

### 第一步：拿代码、装依赖

在项目根目录打开终端：

```bash
npm install
```

再装 Python 依赖（建议在虚拟环境里）：

```bash
cd backend   # 若你在根目录，也可用：pip install -r backend/requirements.txt
pip install -r requirements.txt
```

### 第二步：配置环境变量

复制一份示例配置，改成你自己的（**不要**把真 Key 提交到公开仓库）：

```bash
cp backend/.env.example backend/.env
```

用编辑器打开 `backend/.env`，至少填：

- `ARK_API_KEY`：方舟 API Key  
- `EMBEDDING_ENDPOINT_ID`：Embedding 的 `ep-`  
- `CHAT_VISION_ENDPOINT_ID`：对话/视觉的 `ep-`  

（若你暂时不填，也可以全靠浏览器侧栏里填，见下一步。）

### 第三步：开两个终端，各跑一个服务

**终端 A — 后端**（在项目根目录执行）：

```bash
uvicorn backend.main:app --reload --port 8000
```

看到服务起来后，不要关窗口。

**终端 B — 前端**：

```bash
npm run dev
```

默认浏览器打开 **http://localhost:3000**（或终端里提示的地址）。

### 第四步：侧栏里填「三样东西」

1. 点左下角 **「API 设置」**（齿轮）。
2. 填：**API Key**、**Chat/Vision 接入点 ID**、**Embedding 接入点 ID**（都是方舟控制台里复制的 `ep-`）。
3. 可先点 **「检测模型可用性」**，确认两项都显示可用，再点 **保存**。

配置会存在本机浏览器的 **localStorage** 里，下次打开还在。请求发往后端时，会**优先用这里填的值**；没填的项才用 `backend/.env`。

## 界面怎么用？（按日常顺序）

1. **新建或选中一个项目**（左侧列表）。每个项目的文书、聊天、证据是**分开的**，不会串台。
2. **上传证据**：在证据池区域上传 PDF、Word、zip、图片等。zip 会解压到当前项目目录。上传时若带了 Key 和 Embedding 接入点，后台会尝试**自动做向量入库**（大文件可能要等一会儿）。
3. **上传参考模板**（可选）：传到「模板」入口；生成意见书时会模仿这份模板的结构和语气（模板目录里只取**第一个**文件，建议只放一份）。
4. **中间编辑器**：像写文档一样改；切换项目时会加载该项目已保存的正文；编辑会**防抖自动保存**到服务器。
5. **右侧聊天**：针对**当前项目**提问。AI 会先在向量库里检索片段再回答，适合问「这份合同里写了什么」「有没有提到某某日期」这类问题。

## 常见问题（极简）

| 现象 | 可以试 |
|------|--------|
| 页面说连不上后端 | 确认终端 A 里 **8000** 端口的 uvicorn 在跑，且没有防火墙拦截。 |
| 探测 Embedding 失败 | 确认 Embedding 填的是 **ep-** 接入点，且该接入点绑定的是支持当前 API 的向量模型。 |
| 聊天/生成说缺 Key 或缺接入点 | 侧栏三格保存一遍，或检查 `backend/.env` 是否补齐。 |
| 换了项目，聊天内容变了 | 正常，聊天记录是按**项目**存的。 |
